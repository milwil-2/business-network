About

A B2B trade and trust network: a graph of businesses that buy from, sell to, and supply each other. Each company acts as a buyer, seller, or vendor and gets a trust score calculated from its own data (credit, licenses, age, verification) plus its trade connections (references it has received, owners it shares). Click any company to see how its score breaks down.

Companies also carry trade-role labels (Buyer, Seller, Vendor, Customer) built from their trade edges, so a query like MATCH (b:Buyer) works while each business stays one node.

The data here is generated for the demo. Real company data comes later.

How trust is scored

Creditworthiness30
Trade references20
Verification status20
License validity12
Longevity8
Principal integrity10
A weighted score from 0 to 100. Principal integrity drops to 0 when a company shares an owner with a flagged business, which is the main fraud signal.

Example graph queries (Cypher)

// Fraud ring: principals of >1 flagged company
MATCH (p:Person)-[:PRINCIPAL_OF]->(c:Company)
WHERE c.status = 'flagged'
WITH p, collect(c.label) AS cos, count(c) AS n
WHERE n > 1 RETURN p.label, n, cos ORDER BY n DESC;
// Vendor recommendation (2-hop network effect)
MATCH (me:Company)-[:SELLS_TO]->(b:Company)<-[:SUPPLIES]-(v:Company)
RETURN me.label, b.label AS shared_buyer, v.label AS vendor,
       v.trust_score ORDER BY v.trust_score DESC LIMIT 10;
// Trust hubs (most-referenced companies)
MATCH (ref:Company)-[:GAVE_REFERENCE_FOR]->(c:Company)
RETURN c.label, c.trust_score, count(ref) AS refs
ORDER BY refs DESC LIMIT 10;
// Trust path between two companies
MATCH (a:Company {id:'...'}), (b:Company {id:'...'}),
      p = shortestPath((a)-[*..6]-(b)) RETURN p;
// Most-trusted vendors
MATCH (v:Vendor) RETURN v.label, v.trust_score
ORDER BY v.trust_score DESC LIMIT 10;
